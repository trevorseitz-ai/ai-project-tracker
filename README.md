# ◈ AI Project Tracker

A React app for tracking multiple AI projects across different models and services — with intelligent **Prep**, **Push**, and **Pull** reporter agents that can work autonomously or with a human in the loop.

---

## What It Does

When you're running multiple AI projects across ChatGPT, Gemini, Claude, local models, and autonomous agents, keeping track of progress, blockers, and next steps becomes chaotic. This tracker solves that by:

- Accepting raw updates from **any AI** and normalising them into a consistent schema
- Generating **reporter agents** that plug into your projects and report back automatically
- Running a **Prep agent** that audits a project before a reporter arrives, fixing gaps and writing a handshake config file
- Working with fully **autonomous agents** (no human involvement) and **human-in-loop** projects equally

## The Agent Architecture

```
┌─────────────────────────────────────────────────────┐
│                  AI Project Tracker                  │
│                                                      │
│  ┌──────────┐   ┌──────────┐   ┌─────────────────┐  │
│  │   PREP   │ → │  PUSH /  │ → │   BOARD / LOG   │  │
│  │  AGENT   │   │  PULL    │   │   UPDATE TAB    │  │
│  └──────────┘   └──────────┘   └─────────────────┘  │
└─────────────────────────────────────────────────────┘
       ↓                ↓
  .tracker-config   reporter.py
  .json written      dropped in
  to project         to project
```

### ◈ Prep Agent
Runs **before** any reporter arrives. Audits the project across 6 categories, fixes what it can automatically, and writes a `.tracker-config.json` handshake file. The reporter reads this on arrival — it never has to guess at project name, stack, model, status, or tracker URL.

**Audit categories:**
- Identity — clear project name and purpose
- Documentation — README exists and is meaningful
- Version Control — git initialized, commits are descriptive
- Stack Legibility — dependencies declared, AI model identified
- Status Clarity — current status is determinable, blockers surfaced
- Reporter Readiness — `.tracker-config.json` can be written, TRACKER_URL is set

### ▲ Push Mode Reporter
The tracker **knows** the project and pushes a fully pre-configured Python reporter to it. All config is hardcoded — the receiving agent drops it in and runs it with zero setup. Generates `report_update()` and `report_session()` functions the host agent can call at any time.

### ▼ Pull Mode Reporter
The tracker spawns a reporter that goes **out** to a project it knows nothing about.

- **Autonomous**: inspects README, git log, package.json, requirements.txt, TODO files — builds context, posts first update, all headlessly
- **Human Interview**: live conversational intake inside the app — asks one question at a time, auto-logs first update when it has enough context

## Running It

### Option A — Claude Artifact (no setup)
1. Open [claude.ai](https://claude.ai)
2. Paste the contents of `claude-artifact/App.jsx` into a new message asking Claude to render it as a React artifact
3. Runs immediately — no API key needed, authentication is handled by Claude.ai

### Option B — Self-Hosted (local or deployed)

```bash
cd self-hosted
npm install
cp .env.example .env
# Add your Anthropic API key to .env
npm run dev
# Open http://localhost:3000
```

**Requirements:**
- Node.js 18+
- An [Anthropic API key](https://console.anthropic.com/keys)

**Full documentation:** see [docs/README.md](./docs/README.md) for guides in recommended reading order.

## The `.tracker-config.json` Handshake File

Written by the Prep agent into the project root. Read by any reporter on arrival.

```json
{
  "project_name": "My Agent Project",
  "description": "What the project does",
  "model_used": "GPT-4o",
  "stack": ["Python", "LangChain", "Pinecone"],
  "status": "Active",
  "blockers": [],
  "next_steps": ["Deploy to staging"],
  "tracker_url": "https://your-tracker.example.com/api/update",
  "agent_key": "prep-generated-key",
  "prepped_at": "2026-01-01T00:00:00.000Z",
  "prep_version": "1.0",
  "reporter_mode": "push"
}
```

## Update Schema

All updates — whether pasted manually, generated by a reporter, or sent by an autonomous agent — use this schema:

```json
{
  "type": "feature | tool | daily | blocker | progress",
  "project": "project name",
  "summary": "one clear sentence",
  "detail": "full description",
  "model_used": "AI model or service",
  "status": "Active | Blocked | Stalled | Complete",
  "blockers": ["..."],
  "next_steps": ["..."],
  "confidence": 0.0,
  "missing_fields": ["..."]
}
```

### For Autonomous Agents

POST this schema to your tracker endpoint:

```
POST /api/project-update
Content-Type: application/json
X-Agent-Key: your-agent-key
```

Add this to your agent's system prompt to make it report automatically:

```
After completing each work session, call report_update() with:
- type: feature|tool|daily|blocker|progress
- project: project name
- summary: one sentence
- detail: full description
- model_used: your model name
- status: Active|Blocked|Stalled|Complete
- blockers: list any blockers immediately
- next_steps: list next steps
- confidence: 0.0-1.0

Report blockers immediately. Include uncertain fields in missing_fields.
```

## Reporter System Prompt

Copy this into any AI (ChatGPT, Gemini, Copilot) to make it report in the tracker's schema:

```
You are a project update reporter for an AI project tracker.
When the user describes their work session, parse it and respond ONLY with a JSON object using this exact schema:

{
  "type": "feature|tool|daily|blocker|progress",
  "project": "project name as stated or 'Unknown'",
  "summary": "one clear sentence summary",
  "detail": "full description of the update",
  "model_used": "AI model or service mentioned (or null)",
  "status": "Active|Blocked|Stalled|Complete",
  "blockers": ["list any blockers or empty array"],
  "next_steps": ["list any next steps or empty array"],
  "confidence": 0.0-1.0,
  "missing_fields": ["list any fields you are uncertain about"]
}

Be concise. Do not include markdown, explanation, or backticks. Output raw JSON only.
```
